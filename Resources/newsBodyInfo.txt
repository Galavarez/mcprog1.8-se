Многооконный интерфейс
   
   Можно работать с несколькими дампами одновременно.
   Нажимайте на кнопки   AT88   24CXX   Xerox 01/90  главного окна 
   чтобы открыть новый дамп.
   
Чтение и запись файлов форматов hex и bin.
   
Возможность чтения PonyProg файлов форматов eep, e2p, rom.
   (данная опция не работает для АТ88)
 
Новый hex-редактор
   
   Позволяет редактировать hex-зону и ANSI-зону.
   Шрифт и кодировку можно поменять через Options->Hex editor setting
   
Программирование с автоматической корректировкой дампа 

   Больше не нужно вручную корректировать серийный номер.
   Программа сделает это за вас.
   При нажатии на кнопку Auto на панели главного окна
   открывается окно Auto programmer.
   Подключайте крум выбирайте модель из списка и жмите Program .
   Программа сама откроет файл , запрограммирует крум, увеличит серийный номер, 
   и сохранит файл. 
   Если хотите посмотреть содержимое крума жмите Read.
   Для того чтобы эта опция работала, в одной папке с mcprog.exe
   должна находиться прилагаемая папка dumps.
   В этой папке находится коллекция файлов чистых дампов и файл chipsset.xml
   описывающий эту коллекцию.
   Чистым я называю дамп считанный с крума нового, 
   ни разу не бывавшего в принтере картриджа.
   Вы можете самостоятельно добавлять файлы в коллекцию.
   Сначала нужно добавить файл с чистым дампом в папку dumps.
   Потом занести в файл chipsset.xml запись о новой модели.
   Попробую обьяснить как это сделать.
   Расширение файла xml говорит само за себя.Коллекция описывается XML-документом.
   Откройте этот файл любым текстовым редактором.
   Каждая запись о модели начинается с тега <chip> и заканчивается тегом </chip>.
   Вам нужно вставить запись типа:
   
   <chip>
    <name>24C04 Samsung SCX-4200</name>
    <type>24CXX</type>
    <subtype>24C04</subtype>
    <serialnumtype>1</serialnumtype>
    <templatefile>dumps\\samsungSCX-4200.bin</templatefile>
   </chip>
   
   Тем кто не знаком с XML постараюсь попроще обьяснить.
   
   Элемент XML документа описывается следующим образом
   
   <elem>Content</elem>
   
   <elem> - открывающий тег
   
   </elem> - закрывающий тег
   
   Content - содержимое
   Содержимым может быть текст или другие элементы.
   
   elem - это имя элемента
   
   файл chipsset.xml устроен следующим образом :
   элементы chip содержатся в элементе chipsset и описывают модели
   каждый элемент chip содержит элементы name type subtype serialnumtype templatefile
   
   элемент name содержит название модели которое будет отображаться
   в раскрывающемся списке моделей в окне Auto resetter.
   
   элемент type содержит запись о типе микросхемы крума
   поддерживаемые типы AT88 и 24CXX
   
   элемент subtype описывает подтип миросхемы
   для AT88 это строка описывающая пароль 
   для 24СХХ - размер памяти ( 24C02 или 24С04 , других я не встречал)
  
   элемент serialnumtype описывает способ корректировки дампа
   необходимый для придания ему уникальности.
   No - дамп корректировать не нужно
   1  - нужно корректировать серийный номер находящийся по адресам
        как у большинства крумов например Samsung SCX-4200
   2  - нужно корректировать серийный номер находящийся по адресам
        как у крумов xerox3600 и xerox3635
   
  элемент templatefile описывает путь к файлу начиная от папки с mcprog.exe
  
  
Специальное окно контроля ресеттера.

   При подключении ресеттера последней версии в нижнем левом углу
   главного окна появляется свёрнутое окно Resetter control.
   Не пытайтесь закрыть это окно. Оно закроется автоматически при
   отключении ресеттера.
   В этом окне можно установить напряжение питания крума,
   включить питание в тестовом режиме
   а также посмотреть индивидуальный номер вашего ресеттера 
   и общее количество успешно запрограммированных крумов.
   
Специальное окно для загрузки Firmware.

   При подключении ресеттера с нажатой кнопкой открывается окно Bootloader.
   Не пытайтесь закрыть это окно. Оно закроется автоматически при
   отключении ресеттера.
   Это окно служит для считывания EEPROM контроллера
   и загрузки firmware.
